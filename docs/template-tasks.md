1. **ESQUEMA DE NOMBRAMIENTO PARA ACTIVIDADES:**  
     
   **“Tipo \- Consecutivo planner \- Nombre de la actividad”**  
     
   Los tipos que se van a manejar:  
     
* **FEAT: (Feature)** Se utiliza para el desarrollo de nuevas funcionalidades o capacidades.

  **Cuándo usarlo:** Siempre que estés añadiendo valor nuevo al producto que antes no existía.

  **Ejemplos:** Crear un nuevo botón de exportación, implementar un sistema de login o diseñar una nueva pantalla de perfil de usuario.


* **FIX:** Este tipo se reserva para la resolución de errores o comportamientos no deseados (bugs).

  **Cuándo usarlo:** Cuando algo que ya fue entregado o está en producción no funciona como debería.

  **Ejemplos:** Corregir un error de visualización en la interfaz, arreglar un cálculo matemático erróneo o solucionar una caída del sistema.


* **HOTFIX:** Este término suele ser una variante de "Hotfix" o una "Feature de Urgencia". Representa prioridad máxima.

  **Cuándo usarlo:** Para cambios que no pueden esperar al siguiente ciclo de trabajo o sprint. Debe usarse con moderación para no perder su sentido de urgencia.

  **Ejemplos:** Un parche de seguridad crítico, una funcionalidad legal que debe salir hoy mismo o una reparación urgente que está bloqueando a todo el equipo de ventas.


* **TEST:** Se utiliza para tareas relacionadas con la verificación y el aseguramiento de la calidad.

  **Cuándo usarlo:** Cuando la tarea principal no es construir algo nuevo, sino validar que lo que ya existe funciona correctamente.

  **Ejemplos:** Creación de planes de prueba, ejecución de pruebas manuales, automatización de scripts de prueba o sesiones de QA.
